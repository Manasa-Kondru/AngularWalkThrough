ROUTE GUARDS :
  • To restirct the access to certain components by authentication.
  • We use the Angular Guards to control,whether the user can navigate to some page(route) or not i.e providing authorized access to routes.
  • USES OF ROUTES GUARDS:
  • Asking whether to save before moving away from view.
  • Allowing access to certain parts of the application to specific users.
  • Validating the routing parameters before navigating to route.
  • Fetching some data before you navigate to the route.
  • TYPES OF ROUTE GUARDS IN ANGULAR:
    1. CanActivate:Used to check authentication 'before' entering into that specified component.
    2. CanDeactivate:Used to check authentication 'before' leaving into that specified component.
    3. Resolve:Used to do certain api task before leaving the component.
    4. CanLoad:Used to load only certain modules rather than all.
    5. CanActivateChild:Used to check whether the child components in a route should load or not.
  • To use any route guards we have respectives methods ,which are implemented using service files.
